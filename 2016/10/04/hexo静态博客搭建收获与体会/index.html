title：hexo静态博客搭建收获与体会

静态博客的优劣势

  1. 优势：
  • 提高博客撰写体验
    对于很多人来说，博客系统维护是一件很麻烦的事，而静态博客程序的出现可以避免不必要的系统维护，从而使个人博主们能够更加专注于内容本身的建设。
  • 轻量 程序更安全
    静态博客更加轻便快捷，减轻了服务器的负担，没有数据库，没有后台，完全静态页面，从而大大提高了网站打开的速度。
  • 部署方便 降低建站成本
    由于主流的云存储和代码托管系统都可免费放置静态页面，所以使用静态博客程序进行博客建设可以省下程序托管费用，从而降低建站成本。
  1. 劣势：
  • 需用第三方评论工具
    由于使用静态博客没有数据库，一切评论数据是无法存储的，所以如果博客需要评论功能，那么就必须使用第三方提供的评论工具。
  • 操作相对繁琐 管理相对困难
    由于静态博客没有数据库，所以一切的功能全是手动编写，图片等一些静态资源管理起来也十分麻烦。
  • 不易上手 新手难以使用
    由于管理上的不足，静态博客对新手来说真的难以使用，不懂HTML，JS,CSS根本谈不上使用，发起文章来会十分吃力。

静态博客搭建步骤

说完优劣势之后正式进入博客搭建教程，需要说明的是：以下教程是我在参考网上教程的基础上结合自己在编写过程中遇到的问题的基础上写成的。

  • 因为我们是利用GitHub来搭建博客，说以需要事先注册好GitHub的账号。
    点击注册GitHub账号
    此外还需要下载node.js与git
    点击此处下载node.js
    点击此处下载git（Windows版）

  • 安装hexo
    打开cmd或git bash，执行：npm install -g hexo

  • 创建hexo文件夹
    文件夹的名字是任意的，但此处因为是安装hexo，所以我取名为hexo。
    安装的方式有两种：1. 找一个自己喜欢的路径，在cmd或git bash模式下cd到此路径下，执行命令mkdir hexo即在此路径下创建成功了hexo文件夹。 2. 在’计算机’打开自己喜欢的路径，创建hexo文件夹。这里我建到了E盘下。之后：
    cmd或git bash窗口切换到对应的路径下，然后执行：hexo init

  • 安装依赖
    继续执行：npm install
    此步骤完成后就安装完成了所有依赖。

  • 完成本地安装
    继续在E:\hexo下执行：hexo generate
    继续执行：hexo server
    然后打开浏览器，输入localhost：4000，就可以看到hexo页面，说明博客在本地已经搭建好了。

  • 在GitHub上创建博客仓库：
    1.新建仓库： repository!点击github页面右上角的加号，选择New repository

    2.跳转后如下填写：（其中repository name的格式是‘用户名.github.io’）,如何点击创建仓库（Create repository）至此仓库创建完毕。

  • 创建SSH Keys

  1. 在hexo文件夹下打开git bash，执行如下命令，your_eamil@example.com部分填写自己的邮箱：ssh-keygen -t rsa -C "your_email@example.com"
    然后会出现如下代码Generating public/private rsa key pair. Enter file in which to save the key (/Users/you/.ssh/id_rsa):
    就是让你输入SSH keys要保存在哪里,一般不用改,就直接回车就好了.然后会出现:Enter passphrase (empty for no passphrase): [Type a passphrase] 　 Enter same passphrase again: [Type passphrase again]
    此处注意两点1. 这个密码再输入过程中cmd、或git bash窗口不会有任何显示。2. 这个密码之后会用到，需要记好。

  2. 保存SSH keys:

　　创建成功后,他会提示你SSH keys保存在哪里:Your identification has been saved in /Users/you/.ssh/id_rsa. 　　# Your public key has been saved in /Users/you/.ssh/id_rsa.pub. 　　# The key fingerprint is: 　　# 01:0f:f4:3b:ca:85:d6:17:a1:7d:f0:68:9d:f0:a2:db your_email@example.com

  1. 找到SSH keys:
    　　根据上一步里告诉你的路径,找到保存SSH keys的地方,我的是在 C:\Users\2000104591.ssh
    　　其中 id_rsa.pub 就是SSH keys 如果为了防止以后找不到,可以把他们自己另存到其它地方

  • 为github仓库添加SSH keys
    SSH keys创建好了,我们还要把它添加到仓库里去
    打开创建的仓库,点击’Settings’,再点击左侧的’Deploy keys’:如图：

    点击’Add deploy key’:
    然后把之前创建的id_rsa.pub里的内容复制到Key里去,Title部分随便填. 点击’Add key’
    添加的过程中,还要再输入一次github的密码

  • 测试连接
    回到git bash

执行: ssh -T git@github.com
它可能会出现一些乱七八糟的提示,最后是问你yes/no,就输入yes.
如果是这样子叫你输入密码.那这个密码就是在7.2里面设置的那个密码,就输入一下就好了Enter passphrase for key '/c/Users/2000104591/.ssh/id_rsa':

最后它提示你:Hi, 用户名/用户名.github.io! You've successfully authenticated, but GitHub does notprovide shell access.
这样就算ok了~~~

  • 配置_config.yml文件并发布:
    在 E:\hexo 下,有一个文件叫 _config.yml,打开它,拉到最底下,做如下修改:
    deploy: type: github //改成github repository: https://github.com/jinyanhuan/jinyanhuan.github.io //改成自己的用户名 branch: master　
    配置完以后还是在E:\hexo下执行:hexo generate hexo deploy
    然后访问:http://用户名.github.io/ (用户名改成自己的),就可以看到了.
    注意,每次修改本地文件后，需要 hexo generate 才能保存。每次使用命令时，都要在 E:\hexo 目录下 。

    至此，静态博客就搭建成功了。不过如果你觉得hexo的主题不太好看的话，还可以自己再配置主题，我选择了github上最受欢迎的next主题，因为next主题有十分详细的教程，在此不再多言。

搭建的收获与体会

对于我来说，完成这个项目的经历是艰难的，毕竟我才转到计算机学院一个月，在搭建的过程中几乎每一个步骤都会出问题，所以这个博客反复搭建了几次，耗费了几个晚上才最终成功，但结果看起来也不尽如人意，在hexo主题配置过程中，我遇到了最大的麻烦，尽管我选定的NexT主题有详细的教程，但我仍然没有完全实现NexT主题的功能，所以我的博客看起来十分简陋。具体说来，在NexT主题下，我只是设置了博客名，作者名，头像，个性签名等几个基本功能，多说评论虽然也有设置，但却没有在博客中显现出来，不知为什么。至于社交网站链接，站内搜索，404公益页面，分类，标签，字体，代码高亮主题，打赏等一系列进阶功能，全部都没有能够实现。这些功能，我前后尝试了三遍，都没能够实现，第一次是代码修改完毕，hexo g，hexo d完成之后，在我的博客页面没有显示出来相应的图标（虽然我点击空白部分是可以出来相应的功能的），第二，三次设置完成后，博客页面直接变成了全白，什么都没有显示。而又因为我不知道如何退回到修改前的状态，所以每一次我都得把github仓库删掉，再重头开始。而且又因为我是一名新手，看不太懂cmd中报错的内容（好像是缺什么html文件，然而我并不清楚如何让这些文件出来），事实上我是一步步照着教程修改代码的，然而每次都出问题，这样反复下来就耗费了不少时间。最终，因为我在这个博客上面耗费了太多的时间与精力，以至于打断了我正常的学习计划。不得已，只能建设到这个步骤，当然，这不是最终的结局，等过段时间，我学到了一些前端的知识后再来建设它。

不过细想来，在搭建博客的过程中我收获了很多我知道了hexo是什么，如何在github建仓库，还第一次接触到了Markdown语法写作。不过，最重要的收获是我在这个过程中，明白了一个道理，那就是：要想成为大牛，就要不断地尝试，只有在失败中才能获得成长。

结果并不尽如人意，但过程才是最重要的。在这个过程中，我体会到了计算机的魅力，学到了成长最重要的方法：尝试，有了这些，我相信，我会在以后的路上走得更加轻快。
